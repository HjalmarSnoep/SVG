Animatie oorlogen

SMIL, Web-Animation, CSS-Animation, javascript

Tot nog toe hebben we dingen geanimeerd met javascript. Dit is ook handig als er interactie in je SVG is.

Voor animaties, zijn er echter een aantal alternatieve systemen, SMIL door applie, Web-Animation door Chrome en CSS-Animation dat nooit bedoeld was om SVG mee te animeren, maar SVG is nu eenmaal te stylen met CSS en daar hoort dus ook CSS-animation bij.

Het grote probleem met deze standaarden, is dat niemand het over eens lijkt te kunnen worden. Chrome had SMIL op deprecated staan, maar heeft dat weer ingetrokken na veel druk uit de community die SMIL voorstaat. Web-Animation API wordt niet ondersteund door Edge etc etc. Het is een elllende.
Ook zijn niet alle eigenschappen te animeren.


Daarom ga ik in deze tutorial uit van Javascript. Dit heeft een aantal voordelen.

  • Javascript is zowel voor interactie als animatie geschikt
  • Er zijn goede libraries als greensock en D3 beschikbaar om je te helpen met moeilijke dingen
  • Je hebt geen last van dit soort browser-strijd. Dit omdat Ecmascript (de makers van Javascript) als uitgangspunt hebben: "Don't break the web". Oude versies van javascript blijven het gewoon doen, ook al komt er functionaliteit bij.
Het enige nadeel is, dat Javascript bij echt simpele animaties wat groter uitpakt (meer Kb). Bij de meeste complexere animaties wint javascript weer.
