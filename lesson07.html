Al deze manieren van plaatsen hebben zo hun voordelen en nadelen.
ManierStylen met CSSGebruik externe SVG voor cachingSVG Viewport ControlFallback voor oude browsers
Inline✔❌✔✔
img❌✔❌✔ (alt)
background-image❌✔❌❌ (alleen met js)
extern-object❌✔❌✔
xlink✔✔✔❌ (alleen background-image)

Zo kun je met een object en inline WEL de SVG stylen met css, maar heb je minder controle over bijvoorbeeld de grootte bij het object. Ook is een inline SVG wat verkwistend omdat je geen gebruik kan maken van caching over meerdere pagina's. Zo heb je bij een img of een background img weer de beschikking over allerlei CSS styling, die moeilijker zijn bij standaard SVG, omdat deze zich standaard meer gedraagd als een img met object-fit eigenschappen. Ook met oudere browsers heb je issues met de support van deze verschillende manieren, maar nieuwe browsers ondersteunen ze allemaal erg goed.

Viewport control

Een van de minst begrepen features van een SVG is de alignment property van de Viewport

Ik plaats hier vijf keer dezelfde SVG. Zoals je ziet, probeert SVG standaard de ruimte te vullen met de viewBox.

preserveAspectRatio

Je hebt controle over de manier waarop SVG dit doet met het attribute preserveAspectRatio.
<svg preserveAspectRatio="xMidYMid meet"></svg>

OPDRACHT 1

Ga naar Icon-moon app, selecteer wat icons en genereer een SVG. Plaats de "symbol-defs.svg" file ergens waar je hem kan vinden. Nu kun je ze gebruiken met:
<svg class="icon"><use xlink:href="inspiration/symbol-defs.svg#icon-location"></use></svg>
Zoals hieronder:

Je kunt ze verkleuren met: 
<style>.icon{fill: #cbae82;}</style>
Je plaatst de svg alsof het een letter uit een font is. Door ze allemaal een class te geven, kun je ze gelijk maken van grootte. De iconen worden keurige in de defs van symbol-defs.svg gezet, als symbol. Deze tag is bedoeld voor precies deze functie: het hergebruik van vormen.
