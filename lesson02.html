SVG, HTML en interactiviteit

In deze les wil ik je laten zien dat werken met SVG, css en javascript erg lijkt op werken met HTML, css en javascript.

SVG doet HTML na

HTML en SVG lijken op elkaar. Ik heb eenzelfde 'div' gemaakt in SVG, de SVG staat inline geplaatst in HTML. Je ziet, dat het er vrijwel hetzelfde uitziet. Het is alleen iets anders opgebouwd. Klik op HTML en kijk hoe dit in elkaar zit.

  • Ik heb een svg gemaakt, die erg lijkt op de div.
  • Anders dan bij HTML, zit de styling IN de svg.
  • Ik beschrijf van ALLE elementen waar ze staan en hoe groot ze zijn.

SVG en HTML zijn dus allebei manieren om content etc. op het scherm van je browser te toveren.

SVG is een plaatje, geen pagina..

SVG heeft het uitgangspunt van een plaatje en HTML het uitgangspunt van een pagina. Het zijn allebei Markup-languagues gebaseerd op XML. Dat is goed nieuws, want je kunt het dus al lezen.
Bij SVG geef je al­leen van ele­menten coor­dinaten op, bij HTML valt het standaard in de "pageflow" en weet je dus niet precies waar een element terecht komt. Dat hangt af van de e­le­men­ten eromheen.

In HTML kun je met position: fixed en position: absolute het gedrag van een SVG een beetje nabootsen. Je kunt in SVG geen page­flow hebben!

Maar wat heb ik er dan aan?

HTML is een taal, die is gebaseerd op doosjes (boxjes), rechthoekige dingen. SVG is gebaseerd op curves. Je kunt dus veel vrijere vormen maken dan in HTML. Ik kan bijvoorbeeld een ellips maken met een heel kleine aanpassing in de code:

  • De rectangle in de div is vervangen door een ellips
(Big deal, dat kun je ook met border-radius, toch?)

SVG is een tekentaal en kan daarin meer dan HTML

Kortom, je kunt elementen makkelijk stapelen, en je bent HEEL vrij in de vormen die je maakt.
SVG is een taal om mee te tekenen.

Ook heb je in SVG een heel stel mogelijkheden voor filters, die je in HTML echt niet hebt, maar ook daar komen we later op.

Het punt is voornamelijk: het werkt zoals je gewend bent, je kunt elementen een ID of een CLASS geven en je kunt ze in javascript selecteren met getElementByID, getElementsByClassName of getElementsByTagName, wat dat betreft gedragen SVG elementen en HTML elementen zich hetzelfde.
